BugDIP-93Corrupted DipTimestamp when using 64 bit java client<p>Hi Brice,</p> <p>We have just encountered a problem with the 64 bit version of DIP (5.4.3) when<br/> the DIP publisher is written in Java: The top 32 bits of the long DipTimestamp<br/> value seem to be reported as 0xFFFFFFFF in this case, thus corrupting the<br/> timestamp. The problem seems to be from the Java publisher; with a Java<br/> subscriber in C++ or Java, the same pattern appears.</p> <p>Would it be possible to fix this as quickly as possible? Unfortunately, it has<br/> become a show stopper for some of our monitoring applications. Also<br/> unfortunately, we did not notice it until today, our first stable beams of the year.</p> <p>Thanks,<br/> Bill</p>
